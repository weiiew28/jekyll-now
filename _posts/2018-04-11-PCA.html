Dimension Reduction and whitening

Lots of machine learning models require the features of input data to be independent for effective training and inference. However, real world data do not necessarily satisfy the assumption. For example, in natural image data, nearby pixels are correlated with each other; in speech data, the nearby vowels are also correlated with each other.

One simple mathematical operation, performing PCA, will be able to transform the data such that (1) the dimension of feature space is reduced (2) guarantees the ‘independence’ of features. The simple math behind is the eigen-decomposition structure of the positive definite matrices.

Suppose that we have data points x1,⋯,xnx_1, \cdots, x_{n}x1​,⋯,xn​, where each point xi∈Rdx^{i} \in \mathbb{R}^{d}xi∈Rd has zero mean. The empirical corariance matrix is:

Σn=1n∑i=1nxixiT\Sigma_{n}=\frac{1}{n}\sum_{i=1}^{n} x_{i}x_{i}^{T}Σn​=n1​∑i=1n​xi​xiT​

It is a positive semidefinite matrix, and has the eigendecomposition:

Σn=UΣUT\Sigma_{n}=U\Sigma U^{T}Σn​=UΣUT

with UUU being an orthnormal matrix and Σ=diag(λ1,⋯,λn)\Sigma=diag(\lambda_1,\cdots, \lambda_n)Σ=diag(λ1​,⋯,λn​) being a diagonal matrix satisfying λ1≥λ2⋯≥λn\lambda_1\geq \lambda_2 \cdots \geq \lambda_nλ1​≥λ2​⋯≥λn​.

There are two interpretations of PCA:

  1. Find a kkk dimension subspace VVV such that the projection of nnn points onto VVV are close to the original data points.
  2. Find a kkk dimension subspace such that the projection of nnn points have the maximal variance.

Let V⊂Rd×kV\subset \mathbb{R}^{d\times k}V⊂Rd×k be an orthogonal basis of the subspace we are seeking. Let the data matrix be XXX (∈Rd×n\in \mathbb{R}^{d\times n}∈Rd×n), with xix_ixi​ being its iii-th column. The projection of XXX onto VVV is

ΠV(X)=VVTX\Pi_{V}(X) = VV^{T}XΠV​(X)=VVTX

If we use the basis VVV, then the projected points have coordinate (∈Rk\in \mathbb{R}^{k}∈Rk): VTXV^{T}XVTX.

For the first interpretation, we want to find

min⁡V∣∣VVTX−X∣∣F2\min_{V} ||VV^{T}X-X||_{F}^{2}minV​∣∣VVTX−X∣∣F2​

where we measure the distance by Euclidean distance (ℓ2\ell_2ℓ2​ norm).

For the second interpreation, we want to find

max⁡VTr(VTXXTV)\max_{V} Tr(V^{T}XX^{T}V)maxV​Tr(VTXXTV)

The solution to the above two optimization problems are the same, i.e,

V∗=U1:kV_* = U_{1:k}V∗​=U1:k​

the top kkk eigenvectors of the empirical covariance matrix. For a more detailed and rigorous explanation, I find the first lecture of Banderia very clear. He also discusses some open question related to PCA.

In particular, if we substitute the solution, the new covariance matrtix is:

1nU1:kTXXTU1:k=U1:kTUΣUTU1:k=Σk\frac{1}{n} U_{1:k}^{T}XX^{T} U_{1:k} = U_{1:k}^{T}U\Sigma U^{T}U_{1:k} = \Sigma_{k}n1​U1:kT​XXTU1:k​=U1:kT​UΣUTU1:k​=Σk​

The projected points indeed have 000 convariance between different features (although to be strict, 000 convariance does not in general imply independence).

For the whitening procedure, we want the variance for each component to be 111. Consider the following linear transformation:

Σ−12UTX\Sigma^\frac{-1}{2} U^{T}XΣ2−1​UTX

It is easy to check that:

1n(Σ−12UTX)(Σ−12UTX)T=Σ−12UT(1nXXT)UΣ−12=Σ−12ΣΣ−12=I\frac{1}{n}(\Sigma^\frac{-1}{2} U^{T}X)(\Sigma^\frac{-1}{2} U^{T}X)^{T} =\Sigma^\frac{-1}{2} U^{T} (\frac{1}{n}XX^{T}) U\Sigma^\frac{-1}{2} = \Sigma^\frac{-1}{2}\Sigma \Sigma^\frac{-1}{2} = In1​(Σ2−1​UTX)(Σ2−1​UTX)T=Σ2−1​UT(n1​XXT)UΣ2−1​=Σ2−1​ΣΣ2−1​=I

In this lecture note, it also introduces some practical methods for performing PCA and whitening procedure, eg, preprocessing and smoothing.

Normalization

After understanding the essence of transforming data sets to have independent features and unit variance, it is clear what one of the most popular trick in training the deep neural network, Batch Normalization is doing.

If I am asked to use one sentence to summarize the batch normalization, I will say: it is to develop a simple procedure that integrates whitening-like normalization and optimization step in training the deep neural network.

First of all, why do we want to normalize the output for each layer? I feel the explanation in the paper plausible. Imagine we divide the neural network into 2 parts, upper layers and lower layers. In the training, we feed the upper layer with the inputs generated from the lower layers. However, the inputs from the lower layers will change when the parameters in the lower layer change (during back propagation), and this will cause the distribution of input for the upper layers non stationary. (On the other hand, the input to the lower layers is the training data, and they will not change.)

Secondly, if we want to perform the whitening procedure for a layer ℓ\ellℓ, with ddd inputs. We would need to (1) compute the mean of this ddd dimensional vector from the training data (2) compute the covariance matrix (ddd by ddd) (3) perform SVD and multiply the original input with the computed eigenvector matrix. Let us abbreviate this process by WTWTWT,

x^=WT(x,(x1,⋯,xm))=ΣB−12UBTx\hat{x} = WT(x, (x_1,\cdots,x_m)) = \Sigma_{B}^{-\frac{1}{2}}U_{B}^{T}xx^=WT(x,(x1​,⋯,xm​))=ΣB−21​​UBT​x

where B=(x1,⋯,xm)B = (x_1,\cdots,x_m)B=(x1​,⋯,xm​) is computed from the training/batch data, and ΣB\Sigma_{B}ΣB​, UBU_{B}UB​ are derived from doing SVD on the empirical covariance matrix using BBB. Note that BBB is a function of the parameters for layer ℓ\ellℓ and below.

If we want to compute the gradient with respect to loss LLL, then, it is necessary to compute the following gradients:
1.∂L∂ΣB\frac{\partial L}{\partial \Sigma_{B}}∂ΣB​∂L​
2.∂L∂UB\frac{\partial L}{\partial U_{B}}∂UB​∂L​
3.∂ΣB∂x\frac{\partial \Sigma_{B}}{\partial x}∂x∂ΣB​​
4.∂UB∂x\frac{\partial U_{B}}{\partial x}∂x∂UB​​

Since SVD is not a linear function of each data point, it seems not obvious how to compute 3 and 4 efficiently.

I think this is the reason the authors instead develop a simple whitening procedure in the paper. Instead of making the inputs independent via doing PCA, they just normalize for each component in layer ℓ\ellℓ to have 000 mean and unit variance. To be more precise, for each input iii,

x^i=xi−E(xi)Var(xi)\hat{x}^i = \frac{x^{i}-\mathbb{E}(x^{i})}{\sqrt{Var(x^{i})}}x^i=Var(xi)​xi−E(xi)​

In the context of using batch for estimating the mean and variance,

E(xi)≈μB=1m∑j=1mxji\mathbb{E}(x^i) \approx \mu_{B}= \frac{1}{m}\sum_{j=1}^{m} x_j^iE(xi)≈μB​=m1​∑j=1m​xji​

Var(xi)≈σB2=1m∑j=1m(xji−μB)2Var(x^{i}) \approx \sigma_{B}^2 =\frac{1}{m}\sum_{j=1}^{m}(x_j^i-\mu_{B})^2Var(xi)≈σB2​=m1​∑j=1m​(xji​−μB​)2

x^i=xi−μBσB2\hat{x}^i = \frac{x^{i}-\mu_{B}}{\sqrt{\sigma_{B}^2}}x^i=σB2​​xi−μB​​

where mmm is the batch size.

Thirdly, another important ingredient for the batch normalization procedure is rescale and shift. They reason that if we just do normalization, then the input to the non-linear activation function will `always’ stay in the linear region (given the mean is 000 with unit variance). This somehow weaken the power of non-linear activation. What they do instead is to introduce γi\gamma^iγi and βi\beta^iβi for each input channel:

yi=γix^i+βiy^i = \gamma^i \hat{x}^i+\beta^iyi=γix^i+βi

One final thought: the batch normalization does not really decorrelate the input features although they claim BN can reduce the internal covariance shift. Can we revisit the whitening precedure and design an efficient way to combine decorrelation and optimization concurrently?
